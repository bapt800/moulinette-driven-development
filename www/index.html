Moulinette Driven development

Pourquoi le Moulinette Driven Development ?

- Tout d’abord, cela permet de vérifier que les fonction exposé répondent bien au norme imposé par l’EPITO. Effectivement il leurs est important de vous demander des fonction aux normes singulière, et difficile à comprendre. Donc notre solution vous permet de vérifier le respect des normes rapidement et facilement.
- De plus, le développement vous demandera assurément de reprendre une fonction déjà codé précédemment, le Moulinette Driven Development vous permet de vous assurer de la consistance de votre code instantanément.

Comment faire confiance à notre set de test unitaire clés en main ?

- L’ensemble du projet Moulinette Driven Development est disponible sur GitHub, ainsi il est possible de voir ce que font exactement les test unitaire.
- Vous pouvez vérifié si un faux positif se déclare.
- Nous proposer un test supplémentaire ou une correction.
